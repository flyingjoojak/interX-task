# User Flow — InterX 채용 솔루션

## 전제 조건
- 면접관은 JWT 로그인 후 모든 기능 접근
- 모든 면접관이 모든 후보자 데이터 공유 열람
- 상태 자동 관리: `미분석` `분석중` `분석완료` (시스템)
- 상태 수동 관리: `서류합격` `서류탈락` `면접합격` `면접탈락` `최종합격` `최종탈락` (면접관, 언제든/어느 화면에서든 변경 가능, 되돌리기 가능)

---

## F1. 인증
```
/login → 이메일+비밀번호 입력 → JWT 발급 → /home
```

---

## F2. 후보자 등록 + 분석 시작
```
/home → [새 후보자] 버튼 → /candidates/new

입력 폼:
  - 이름, 생년(연도), 지원포지션, 면접관메모
  - 이력서 업로드 존: PDF/JPG/PNG/PPT/PPTX (1개)
  - 포트폴리오 업로드 존: PDF/JPG/PNG (1개)
    └─ "PPT/PPTX 미지원. PDF 형식을 권장합니다" 안내 표시
  - 조건: 둘 중 하나 이상 업로드 필수

[등록+분석시작] → candidates 생성 + 분석 백그라운드 실행
  → /candidates/{id} (분석 진행 중 상태로 진입)
```

---

## F3. 사전 분석 진행 중
```
/candidates/{id} (current_step ≠ 완료)

표시:
  - 후보자 기본정보
  - 단계별 진행률 바: OCR → 정보추출 → 가치매핑 → 모순탐지 → 질문생성
  - 예상 남은 시간 (단계별 평균 소요시간 기반)
  - 프론트 2초 폴링 → GET /candidates/{id}/analysis/progress

면접관 이탈 가능 → 백그라운드 계속 진행
완료 시 → 동일 URL에서 리포트 화면으로 전환
```

---

## F4. 사전 분석 리포트 (서류 심사)
```
/candidates/{id} (current_step = 완료)

레이아웃 (스크롤):
  [헤더]
    후보자명 | 포지션 | 상태 드롭다운 | [면접 시작] | [PDF 출력] | [파일 재업로드]

  [Section 1 — 종합]
    종합 요약 텍스트
    12가지 가치 레이더 차트
    문서 신뢰도 점수 (0~100)

  [Section 2 — 가치별 상세]
    12개 항목 리스트: 가치명 | 점수 바 | 숫자
    항목 클릭 → EvidencePanel 슬라이드 오픈 (AI 분석 근거 + 이력서 인용)

  [Section 3 — 모순/허점]
    [{source_a} vs {source_b}: {description}] 형태 리스트
    severity 표시 (high/medium/low)

  [Section 4 — 사전 압박 질문]
    생성된 질문 카드 목록
    각 카드: 질문 | 근거 가치 | 탐지 근거
```

---

## F5. 파일 재업로드 (분석 재시작)
```
리포트 화면 → [파일 재업로드]
  → 확인 모달: "기존 분석 결과와 면접 세션이 초기화됩니다. 계속하시겠습니까?"
  → 확인 → 분석 + 세션 전체 삭제 → 파일 교체 업로드
  → [재분석 시작] → F3으로 재진입
```

---

## F6. 실시간 면접
```
리포트 화면 → [면접 시작]
  → 기존 세션 있음: "이어서 진행 / 새로 시작" 모달
      새로 시작 선택 → 기존 세션 Q&A 초기화 + 새 세션 생성
  → /candidates/{id}/interview

화면 레이아웃:
┌─────────────────────┬────────────────────────────────────┐
│ 좌측 (고정)          │ 우측 (스크롤)                       │
│                     │                                    │
│ 후보자 기본정보       │ Q&A 타임라인 (채팅형, 시간순 선형)   │
│ 가치 점수 요약        │ [Q] 질문 텍스트                    │
│ 모순 탐지 목록        │ [A] 답변 텍스트                    │
│                     │     ↳ 생성된 꼬리질문들 (접힘/펼침) │
│ ── 사전 질문 ──      │                                    │
│ ▶ 질문A (미사용)     │ ── 현재 입력 영역 ──               │
│ ✓ 질문B (사용됨)     │ 현재 질문: [표시]                  │
│ ▶ 질문C              │ 후보자 답변: [텍스트에어리어]       │
│                     │                      [분석] 버튼   │
│ [커스텀 질문 입력]   │                                    │
│                     │ [꼬리질문 패널 — 토글]              │
│                     │   FU1: ... [선택]                  │
│                     │   FU2: ... [선택]                  │
│                     │   FU3: ... [선택]                  │
└─────────────────────┴────────────────────────────────────┘
```

**면접 인터랙션 사이클:**
```
1. 좌측 사전질문 클릭 OR 커스텀 질문 입력
   → 우측 현재 질문으로 활성화
2. 후보자 답변 타이핑 → [분석] 클릭
3. 꼬리질문 패널 오픈 (3~5개 생성)
4. 면접관이 꼬리질문 선택 → 1번으로 반복
   OR 다른 사전질문/커스텀질문으로 이동 → 1번으로 반복
```

세션 자동 저장. 이탈 후 재진입 시 F6 시작 시점 모달로 처리.

## F7. PDF 출력
```
리포트 화면 OR 면접 화면 → [PDF 출력]
  → GET /candidates/{id}/report/pdf
  → Playwright 서버 렌더링 → 통합 PDF 다운로드
  포함: 사전 분석 리포트 전체 + 면접 세션 Q&A 전체
```

---

## F8. 후보자 비교 (홈)
```
/home

후보자 카드 목록:
  이름 | 포지션 | 상태 배지 | 12가치 평균점수 | 문서 신뢰도
  → 카드 나열 자체가 비교 뷰 (별도 비교 화면 없음)
  → 상태 필터로 특정 단계 후보자만 조회 가능
  → 카드 클릭 → /candidates/{id}
```

---

## F9. 후보자 삭제
```
홈 or 리포트 화면 → [삭제]
  → 확인 모달
  → 확인 → 후보자 + 업로드 파일 + 분석 + 면접 세션 + Q&A 전부 삭제
  → /home 으로 이동
```
