## 5. Chart E — 토큰 사용량 실측 (슬라이드 12) 【측정 후 채움】

**측정 방법**: 주재홍 이력서 1회 분석 후 아래 API 호출.
```
GET /api/candidates/<candidate_id>/cost
```

**수집할 값 (JSON 필드)**:
- `total.calls`
- `total.input_tokens`
- `total.output_tokens`
- `total.cost_usd`
- `by_phase.analysis.cost_usd`, `by_phase.interview.cost_usd`
- `by_step.OCR.{input_tokens, output_tokens, cost_usd}`
- `by_step.추출.{...}`
- `by_step.가치매핑.{...}`
- `by_step.모순탐지.{...}`
- `by_step.질문생성.{...}`

**차트 구성**:
- **Bar E-1** — step별 input/output 토큰 누적 바 (5개 step, stacked: input 회색 + output #ff8000).
- **Bar E-2** — step별 USD 비용 (수평 바, 단일 주황).
- **KPI 박스** — `1회 분석 총 USD`, `총 토큰`, `호출 수`, `가장 비싼 step`.

**Placeholder (측정 전 초안용)**:
```
총 비용: $0.XXXX  (측정 후 교체)
총 토큰: X,XXX input + X,XXX output
총 호출: N회
비싼 순: 가치매핑 > OCR > 질문생성 > 모순탐지 > 추출  (예상)
```
