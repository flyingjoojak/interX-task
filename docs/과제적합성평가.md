# InterX MVP — 과제 적합성 평가

작성일: 2026-04-19

현재 프로젝트가 주어진 3개 주과제와 가이드 항목에 얼마나 부합하는지 정리한 문서입니다. 각 항목은 **충족 / 부분 충족 / 미충족** 판정과 근거 파일·위치를 함께 기재합니다.

---

## 1. 이력서/포트폴리오 → 12가치 데이터 파이프라인 — **충족**

### 요구사항
> 비정형 데이터(PDF 등)를 OCR/파싱하여 구조화된 데이터(JSON/DB)로 변환.
> 지원자의 특정 경험이 인터엑스의 어떤 가치(예: 강박적 호기심, 미래낙관적 도전 등)에 해당하는지 AI로 분류하는 프롬프트 엔지니어링 수행.
> 결과물: 텍스트에서 추출된 데이터와 핵심가치 매칭 결과가 담긴 구조화된 데이터 셋(JSON 등) 및 기술 기술서.

### 충족 근거
- **비정형 → 구조화 변환**
  - `backend/services/ocr_service.py` — Claude Vision 기반 OCR. PDF/JPG/PNG를 이미지 블록으로 나눠 읽기 순서(위→아래, 왼쪽→오른쪽, 2-컬럼/프로젝트 묶음 유지)까지 지시한 프롬프트로 텍스트 복원.
  - `backend/agents/analysis_graph.py` 의 `parse_documents` 노드 — OCR 텍스트를 `STRUCTURED_EXTRACTION_PROMPT`(prompts.py:25)로 돌려 `career / education / skills / achievements / certifications / projects` JSON으로 변환.
  - 결과는 `Analysis.structured_data` 컬럼에 JSON 직렬화되어 저장.

- **12가지 핵심가치 매칭**
  - `backend/agents/prompts.py` 상단 `INTERX_VALUES` 에 12가지 가치 정의 상수화.
  - `VALUE_SCORING_PROMPT`(prompts.py:68) 로 가치별 `score / evidence / examples` 구조로 평가 — 추상적 자기서술은 증거로 인정하지 않고 행동·수치·결과만 채택하도록 강제.
  - 결과는 `Analysis.values_scores` 에 저장, 프론트엔드 `ValueRadarChart`/`ValueList`/`EvidencePanel` 에서 가치 클릭 시 원문 인용과 함께 시각화.

- **결과물 확인 수단**
  - `GET /api/candidates/{id}/analysis` — 구조화 데이터와 가치 매칭 결과를 통째로 JSON 반환.
  - `GET /api/candidates/{id}/debug/raw` — 문서 OCR 원문과 구조화 데이터를 묶어 반환 (프론트 `RawDataViewer` 에서 탭 UI로 즉시 비교 가능).

## 2. 압박 면접 질문 생성기 — **충족**

### 요구사항
> 후보자의 답변 내용을 바탕으로 사실관계의 모순을 찾아내는 추론 로직 설계(LangGraph 등 활용 권장).
> 후보자가 답변하기 까다로운 고차원적인 '꼬리 질문(Probing Question)' 생성 프롬프트 체인 구축.
> 결과물: 시나리오별 입력 데이터에 따른 AI 에이전트의 질문 생성 로직 및 실제 질문 샘플 리포트.

### 충족 근거
- **모순 탐지 추론**
  - `CONTRADICTION_DETECTION_PROMPT`(prompts.py:137) — 날짜/기간, 직책/역할, 성과 수치, 기술 스택 4축으로 이력서 내부 + 이력서↔포트폴리오 교차 모순을 탐지. severity `high / medium / low` 분류.
  - `Analysis.contradictions` 에 저장, 프론트 `ContradictionList` 에서 출처·설명·심각도 함께 노출.

- **LangGraph 체인** (권장 사항 반영)
  - `backend/agents/analysis_graph.py` — 분석 파이프라인을 `parse_documents → extract_structured → score_values → detect_contradictions → generate_preemptive_questions → summarize` 로 단계화.
  - `backend/agents/interview_graph.py` — 실시간 면접 꼬리질문 체인을 `prepare_context → analyze_answer → generate_followups → rank_and_filter` 4노드 StateGraph로 구성.
    - `prepare_context`: 후보자의 구조화 데이터 + 저점(40점 미만) 가치 + 최근 답변 이력 3개 로딩.
    - `analyze_answer`: 답변의 모호성/불일치/과장 여부를 먼저 짚음.
    - `generate_followups`: 위 분석을 근거로 꼬리질문 후보 생성.
    - `rank_and_filter`: priority 기준 정렬, 상위 5개만 채택.

- **사전 압박 질문 (배치 생성)**
  - `PREEMPTIVE_QUESTIONS_PROMPT`(prompts.py:178) — 모순 + 저점 가치 기반으로 개방형 질문 금지, 반드시 수치·시점·결정자를 묻는 형태로 생성하도록 제약.

- **실시간 꼬리질문 (면접 중 생성)**
  - `INTERVIEW_FOLLOWUP_PROMPT`(prompts.py:208) — 이번 답변에서 가장 취약한 지점을 파고드는 3~5개 질문을 생성.
  - `question_source`(pregenerated / custom / followup)를 프롬프트 입력에 포함해, custom 질문이든 꼬리질문의 꼬리질문이든 생성을 생략하지 않도록 강제.
  - `services/followup_worker.py` — 단일 워커 기반 asyncio.Queue 로 LLM 호출을 직렬화. 동시에 여러 답변이 제출돼도 꼬이지 않음.

- **결과물 확인 수단**
  - `RawDataViewer` 의 JSON 탭에서 `preemptive_questions` 목록을 원문 인용과 함께 확인.
  - 실제 면접 화면(`/candidates/{id}/interview`)에서 사전 질문 선택 → 답변 → 꼬리질문 자동 생성 흐름이 그대로 시연 가능.

## 3. AX 기반 채용 관리 대시보드 — **충족**

### 요구사항
> 분석 결과(가상 데이터 활용 가능)를 시각화하여 보여주는 웹 애플리케이션 구축(FastAPI + Streamlit/React 등).
> AI가 분석한 '사전 분석 보고서'를 PDF나 화면 상에 인터랙티브하게 출력하는 기능 구현.
> 결과물: 실제 동작하는 웹 프로토타입 URL 또는 소스코드 및 구동 영상.

### 충족 근거
- **풀스택 구성**
  - 백엔드: FastAPI (port 8102) — `backend/main.py` 진입점, 라우터 분리 (auth / candidates / documents / analysis / interview).
  - 프론트: Next.js 14 App Router + TypeScript + Tailwind (port 3102).

- **인터랙티브 시각화**
  - `ValueRadarChart` — 12가치 점수를 레이더 차트로 즉시 비교.
  - `ValueList` — 클릭 시 선택 상태 동기화, `EvidencePanel` 슬라이드아웃에서 원문 증거/예시 문장 노출.
  - `ContradictionList` — 심각도 색상 강조 + 출처 양쪽 표시.
  - `PreemptiveQuestions` — 근거(`basis`)와 연결 가치(`target_value`)를 함께 표시.
  - `StatusSelector` — 미분석/분석중/합격/탈락 등 9단계 후보 상태를 드롭다운으로 즉시 변경.
  - `ProgressBar` — 분석 진행 단계와 예상 잔여 시간을 표시.

- **PDF 내보내기**
  - `services/pdf_export.py` — reportlab 기반 서버 사이드 생성. 한글 폰트(Malgun Gothic) 자동 등록. 후보자 정보 / 문서 신뢰도 / 12가치 표 / 모순 / 사전 질문 / 구조화 데이터 전체 포함.
  - `GET /api/candidates/{id}/report/pdf` 로 다운로드, 프론트 `리포트 페이지` 에서 버튼 클릭만으로 받기.

- **면접 관리 흐름**
  - 후보자 등록 → 문서 업로드 → 분석 실행 → 리포트 확인 → 면접 진행(사전 질문 + 실시간 꼬리질문 + 직접 질문 추가) → 상태 변경까지 단일 대시보드에서 수행.

## 가이드 항목(엔지니어링 깊이) — **부분 충족**

### 요구사항
> 단순히 API를 호출하는 코드가 아니라, Hallucination 방지 로직이나 비용 최적화(Token 사용량 관리) 등 엔지니어링 고민이 담겼는지.
> Cursor, Windsurf 등 최신 AI 도구 활용을 적극 활용했는지.

### 반영된 부분

- **Hallucination 방지**
  - 프롬프트 레벨 제약: "원문에 없는 정보는 추측하지 말 것", "추상적 자기서술은 증거로 인정하지 않음", "원문 문장 그대로 인용" 등 반복 명시.
  - **근거 필드 강제**: 12가치(`evidence`/`examples`), 사전질문(`basis`), 꼬리질문(`reasoning`) 모두 근거를 구조화 데이터로 요구 → 생성 후 원문 대조 가능.
  - `DOC_RELIABILITY_PROMPT` — 문서 자체 신뢰도를 일치도/신뢰도/완성도 3축 100점으로 정량화.
  - `RawDataViewer` — OCR 원문 ↔ 구조화 JSON 을 사람이 직접 비교 검증할 수 있는 UI.

- **비용·안정성**
  - `call_claude_json` — JSON 파싱 실패 시 2회까지 재시도 (analysis_runner.py:69).
  - `API_SLEEP_SECONDS` — 호출 간 간격으로 rate limit 완충.
  - 단계별 `max_tokens` 차등: 추출 16384, 꼬리질문 2048 등 과다 할당 방지.
  - **꼬리질문 비동기 단일 워커 큐** (`services/followup_worker.py`) — 동시 답변 제출에도 LLM 호출이 직렬로 흐르며 중복/경합 차단.
  - Phase 단계 상태(`current_step`) 저장 → 중간 실패 시 재분석 재개 가능.

- **참고 (비공개 메모)**
  - 분석 1회당 Anthropic API 비용 추정: $0.15–0.30 / 꼬리질문 1회: $0.03–0.05 범위.

### 보강 여지

- **토큰 사용량 계측 부재** — 현재는 대략치 메모만 존재. Anthropic 응답의 `usage.input_tokens / output_tokens` 를 DB에 적재해 후보자별 실제 비용 리포트로 가공하면 "비용 최적화 고민"을 직접 수치로 증명할 수 있음.
- **Hallucination 자가 검증 루프 부재** — 생성된 `evidence` 문자열이 실제 OCR 원문에 포함되는지 substring/유사도 체크하는 self-verify 단계를 추가하면 출력 검증 축이 명시적으로 드러남.
- **테스트 부재** — 프롬프트 회귀 테스트(고정 입력 → 필수 키 존재)만이라도 추가하면 과제 제출 품질 상승.
- **AI 도구 활용 흔적** — Cursor / Windsurf / Claude Code 사용 내역은 소스에 남지 않음. README 또는 제출 보고서에 "어떤 모듈을 어떤 도구로 생성·리팩터했는지" 명시 권장.

## 요약 판정

| 항목 | 판정 | 비고 |
| --- | --- | --- |
| 1. 데이터 파이프라인 (OCR → 구조화 → 12가치) | 충족 | Claude Vision + 프롬프트 체인 |
| 2. 압박 면접 질문 생성기 (모순 + 꼬리질문) | 충족 | LangGraph 두 개 (analysis, interview) |
| 3. AX 대시보드 (시각화 + PDF) | 충족 | FastAPI + Next.js + reportlab |
| 가이드 (Hallucination 방지 / 비용 최적화) | 부분 충족 | 토큰 계측·자가 검증·테스트 보강 권장 |
| 가이드 (최신 AI 도구 활용) | 미검증 | 소스상 흔적 없음 → 문서로 보완 필요 |

3개 주과제는 모두 충족. LangGraph 권장 사항도 두 개 그래프에 반영됨. "엔지니어링 깊이"를 더 강조하려면 **토큰 사용량 계측**과 **evidence 자가 검증 루프** 두 가지를 추가하는 것이 가장 효과적.
